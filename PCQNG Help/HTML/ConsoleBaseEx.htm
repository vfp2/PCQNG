Console (VC++ 7.0) Basic Example

This example shows how to use the PCQNG ActiveX interface to get random 32-bit integers, uniform numbers [0,1), and normally distributed numbers using the ComScire PCQNG True Random Number Generator.

Click on for a screen shot of each step.
Step 1 Create a new Console Application
Start a new Visual C++ project. Select "Win32 Project" as the template. Name the project and click OK.
In the Win32 Application Wizard click on "Application Settings" on the side bar, then select "Console Application" as the Application Type and click on "Finish".

Step 2 Create access to PCQNG ActiveX control
Open Class View (from the main menu click on "View", then select "Class View").
In Class View, click on the boxed plus next to your project name to expand Class View.
Expand Global Functions and Variables and double click on "_tmain(int argc, _TCHAR* argv[])". This will open the console code.
Add an "#import" line to the QWPCQNG.dll just below the "#include" statements (note: the path to QWPCQNG.dll might differ in your configuration):

#include "stdafx.h"

// #import directive using full path to "QWPCQNG.dll"
// "no_namespace" avoids the need to use namespace "QWPCQNG"
#import "C:\WINDOWS\system32\QWPCQNG.dll" no_namespace named_guids

int _tmain(int argc, _TCHAR* argv[])
{
  return 0;
}
Step 3 Add code to access and display PCQNG data
Add code to _tmain as follows:
 
int _tmain(int argc, _TCHAR* argv[])
{
  // Declare a smart pointer to PCQNG control
  IPCQNGPtr pPCQNG = 0;

  // Create instance of PCQNG control
  CoInitialize(NULL);
  pPCQNG.CreateInstance(CLSID_PCQNG);

  pPCQNG->Open(); // Open PCQNG

  // Get and print a uniform, normal, and 32 bit integer
  cout << "Uniform: " << pPCQNG->RandUniform << endl;
  cout << "Normal: " << pPCQNG->RandNormal << endl;
  cout << "Int32: " << pPCQNG->RandInt32 << endl << endl;

  pPCQNG->Close(); // Close PCQNG

  // Wait for a keypress
  cout << "A Key to Quit" << endl;
  while (!kbhit());

  return 0;
}
Step 4 Compile and run
Click on the Debug menu and select "Start Without Debugging" to run the application. If the compiler finds errors, check to make sure the code appears exactly as shown in step 3.
