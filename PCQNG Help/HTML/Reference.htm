PCQNG ActiveX Reference

The PCQNG is accessed using an ActiveX control which has six methods to control the PCQNG device and four properties to access the random data.

Programming Notes:
Call the Open() method before making any other calls to the PCQNG control. Open() will attempt to acquire the resources necessary for running the PCQNG. The Open() call may return a PCQNG_E_DEVICE_UNAVAILABLE exception, if the PCQNG is being used by another program on the same machine. Only a single instance of PCQNG may be running on a single machine at a time. If another call is made before Open() is called, a PCQNG_E_DEVICE_NOT_OPEN exception will be thrown. Please call the Close() method to release the PCQNG to the system after the PCQNG is no longer needed by your program. This allows other Programs to gain access to the PCQNG resource.

The PCQNG provides custom HRESULT codes on every method or property call. HRESULT codes provide exception/error detail. See Error Handling on catching the HRESULT codes.

During a programming session, a programming environment typically sets the Ambient Properties for an ActiveX control to design mode. In design mode the control will return default values for RandInt32 (123456789), RandUniform (0.123456789), RandNormal (1.23456789) and RandBytes (1234567890123...). If the PCQNG control returns these values at run-time, please make sure the User Mode Ambient Property is set to TRUE at run-time. This should normally not be an issue, however.
